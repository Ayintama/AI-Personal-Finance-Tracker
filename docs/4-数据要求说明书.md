# 数据要求说明书

**项目名称**：AI 个人财务管理系统

---

## 1. 数据内容与来源

本系统管理的数据按数据类别划分为：

| 数据类别 | 来源 | 敏感度 | 说明 |
|---------|------|--------|------|
| 用户账号 | 用户注册时录入 | 高 | 含用户名、密码哈希、邮箱 |
| 分类数据 | 系统预置 + 用户自定义 | 低 | 收入/支出分类 |
| 账单数据 | 用户记账录入 | 高 | 含金额、分类、时间、备注 |
| 预算数据 | 用户设置 | 中 | 含月份、金额、可选分类 |
| Token | 后端生成 | 高 | JWT 格式，有效期 7 天 |

| 字段名 | 类型 | 允许空 | 是否主键 | 说明 |

### 2.4 budgets 表（预算表）

| 字段名 | 类型 | 允许空 | 是否主键 | 说明 |
|--------|------|--------|----------|------|
| id | BIGINT | 否 | 是 | 预算 id |

## 3. 数据完整性

### 3.1 实体完整性

- 所有表使用 BIGINT 自增主键
- 关键字段使用 NOT NULL 约束

### 3.2 参照完整性

- `transactions.user_id` → `users.id`
- `transactions.category_id` → `categories.id`
- `budgets.user_id` → `users.id`
- `budgets.category_id` → `categories.id` （可为空）

> 注：为简化部署，当前版本暂未声明外键约束，数据一致性由应用层保证。

### 3.3 领域完整性

| 约束 | 说明 |
|------|------|
| 用户名长度 | 3-50 字符 |
| 密码长度 | ≥ 6 字符 |
| 金额范围 | > 0 且 ≤ 99,999,999.99 |
| transaction_type | 仅允许 "income" / "expense" |
| month 格式 | YYYY-MM，长度 7 |

### 3.4 用户数据隔离

- 所有查询均使用 `user_id = ?` 条件
- 删除/更新前二次校验 `user_id` 归属
- 未登录请求由 JWT 中间件直接拦截并返回 401

## 6. 数据安全

### 6.1 密码安全

- 使用 `bcrypt::DEFAULT_COST`（cost = 12）对密码进行加盐哈希
- 不存储明文密码
- 数据库中仅保存 `password_hash`

### 6.2 Token 安全

- 使用 HS256 算法签发 JWT
- Token 中仅携带 `sub (user id)`、`username`、`iat`、`exp`
- Token 有效期：7 天（604,800 秒）
- 前端 localStorage 存储 Token，每次请求放入 `Authorization: Bearer <token>`

### 6.3 防 SQL 注入

- 全部 SQL 使用参数绑定（`?` 占位符）
- 禁止通过字符串拼接构造 SQL

## 7. 数据备份与恢复

### 7.1 备份策略

| 周期 | 操作 | 保留期 |
|------|------|--------|
| 每日 | `mysqldump -u user -p ai_finance > backup_YYYYMMDD.sql` | 7 天 |
| 每周 | 与每日相同，额外归档 | 30 天 |

### 7.2 恢复步骤

```bash
# 1) 停止应用访问
# 2) 执行 SQL 脚本恢复数据
mysql -u root -p ai_finance < backup_YYYYMMDD.sql
```
