# 用户手册

**软件名称**：AI 个人财务管理系统（AI Personal Finance Tracker）


---

## 1. 软件简介

本系统是一款轻量的 Web 个人财务管理工具，面向普通用户、家庭财务管理者，提供：

- 账户注册与登录
- 收支账单记账（增删改查）
- 按月份查看收入/支出统计
- 设置月度总预算与分类预算，实时跟踪进度
- 生成带消费建议的月度 AI 报告

本系统分为两种使用模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 在线模式 | 数据存在 MySQL 数据库，真实持久化 | 正式使用、多人部署 |
| 演示模式 | 数据存储在浏览器本地 localStorage | 快速体验 / 无后端环境 |

---

## 2. 运行环境

| 项目 | 最低要求 |
|------|---------|
| 操作系统 | Windows 10 / macOS 10.14 / Linux |
| 浏览器 | Chrome 100+、Edge 100+、Firefox 100+ |
| 屏幕分辨率 | 1024 × 768 及以上 |
| 网络 | 登录与在线模式需要网络访问后端；演示模式可离线 |

> 若使用在线模式，需要后端服务已启动（默认 http://127.0.0.1:8080），并在浏览器打开 `login.html`。

---

## 3. 启动步骤

### 3.1 第一次使用（在线模式）

1. **启动后端**（由系统管理员完成）：
   ```bash
   cd backend
   cargo run
   ```
   启动成功会显示 `Starting server at http://127.0.0.1:8080`

2. **打开登录页**
   双击 `frontend/login.html` 或在浏览器打开该文件。

3. **注册账号**
   点击登录页的"注册"标签 → 输入用户名、密码、邮箱 → 点击"注册" → 自动跳转到主页。

### 3.2 第一次使用（演示模式）

无需后端服务。直接：

1. 打开 `frontend/login.html`
2. 使用预置账号登录：
   - 用户名：`test`
   - 密码：`test123456`
3. 登录成功后可直接使用，数据保存在浏览器 localStorage。

### 3.3 后续登录

每次只需打开 `login.html` 输入用户名和密码即可进入系统。

**登录失败的常见原因**：

| 现象 | 原因 | 解决办法 |
|------|------|---------|
| "后端服务未启动" | Rust 后端未运行 | 启动 `cargo run` |
| "用户名或密码错误" | 账号不存在或密码错误 | 确认账号密码；可重新注册新账号 |

| "密码至少 6 位" | 注册时密码太短 | 输入更长密码 |

## 5. 主界面介绍

登录成功后，进入主页面，顶部为 4 个标签页（Tab）：

```
┌───────────────────────────────────────────────────────┐
│ AI 记账本   仪表盘 | 账单 | 预算 | AI 报告    test ▼ 退出│
├───────────────────────────────────────────────────────┤
│                                                       │
│   仪表盘（默认显示页）                                  │
│   ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐    │
│   │ 本月收入│  │ 本月支出│  │ 本月结余│  │ 预算使用│    │
│   │ ￥8500│  │ ￥1500│  │ ￥7000│  │  50%   │    │
│   └────────┘  └────────┘  └────────┘  └────────┘    │
│                                                       │
│   [ 分类占比饼图 ]     [ 近期账单列表 ]                │
│                                                       │
└───────────────────────────────────────────────────────┘
```

### 5.1 Tab 1：仪表盘

显示内容：

- **本月收入**：当月所有收入账单合计
- **本月支出**：当月所有支出账单合计
- **本月结余**：收入 − 支出
- **预算使用**：本月总预算的百分比；未设置预算时显示"—"
- **分类占比饼图**：当月支出按分类的占比分布
- **近期账单**：最近新增的若干账单

**切换月份**：点击仪表盘顶部的月份选择器，可查看任意历史月份的数据。

---

### 5.2 Tab 2：账单管理

页面显示当前月份的账单列表，表格字段：

| 列 | 说明 |
|----|------|
| 发生时间 | YYYY-MM-DD HH:MM |
| 类型 | 收入（绿色）或支出（红色） |
| 分类 | 如"餐饮""交通""工资"等 |
| 金额 | 记账金额，保留两位小数 |
| 备注 | 自定义备注，如"午餐""演唱会门票" |
| 操作 | 编辑 / 删除 |

**功能操作**：

**① 筛选**

- 月份选择器：切换到想看的月份（默认当月）
- 类型选择：全部 / 收入 / 支出

**② 分页**

- 每页显示 20 条，可翻页（首页/上一页/下一页）

**③ 新增账单**

1. 点击"新增账单"按钮 → 弹出表单
2. 选择"类型"（收入或支出）
3. 选择"分类"（下拉菜单）
4. 输入"金额"（正数）
5. 选择发生时间（默认当前时间）
6. 填写备注（可选）
7. 点击"保存"

**④ 编辑账单**

1. 在账单列表中点击"编辑"
2. 弹出与新增相同的表单，修改后保存

**⑤ 删除账单**

1. 在账单列表中点击"删除"
2. 弹出确认对话框，确认后删除

> ⚠️ 删除后无法恢复，请谨慎操作。

---

### 5.3 Tab 3：预算管理

**① 设置总预算**

1. 点击"设置总预算"按钮
2. 选择月份
3. 输入预算金额（例如 3000）
4. 点击"保存"

保存后，仪表盘的"预算使用"会更新。

**② 设置分类预算**

1. 点击"设置分类预算"
2. 选择月份、分类、金额
3. 点击"保存"

**③ 预算列表**

表格字段：

| 列 | 说明 |
|----|------|
| 分类 | 总预算或分类名 |
| 预算金额 | 设置的预算额度 |
| 已用金额 | 截止到今日已花费金额 |
| 剩余 | 预算 − 已用 |
| 进度 | 百分比进度条（0%~100%，超出时显示红色） |
| 操作 | 删除 |

**预算使用提示**：
- 使用率 ≤ 80%：显示绿色
- 80% < 使用率 ≤ 100%：显示黄色
- 使用率 > 100%：显示红色，并在 AI 报告中提醒超支

---

### 5.4 Tab 4：AI 报告

点击"生成报告"后，系统会分析该月份账单数据并生成一份报告，内容包括：

- **月度财务概览**：收入、支出、结余
- **最大支出分类**：本月消费占比最高的分类
- **预算使用率**：本月预算使用情况
- **AI 建议**：系统根据数据分析给出 1~4 条中文建议

示例报告：

```
2026 年 6 月 财务概览
收入：￥8,500.00
支出：￥1,500.00
结余：￥7,000.00
预算使用：50%（1500 / 3000）

最高支出分类：餐饮

AI 建议：
  · 本月最大支出类别为"餐饮"，建议检查是否有非必要消费。
  · 预算使用进度 50%，整体可控。
  · 本月结余为正，可考虑将部分资金用于储蓄或学习投入。
```

## 6. 智能分类说明

当新增账单填写备注时，系统会尝试根据关键词自动推断分类（在后端模式下可用）。常见关键词：

| 关键词示例 | 匹配分类 |
|-----------|---------|
| 午餐、晚餐、外卖、奶茶、咖啡、食堂 | 餐饮 |
| 地铁、打车、滴滴、公交、高铁、机票、加油 | 交通 |
| 淘宝、京东、购物、衣服、超市、拼多多 | 购物 |
| 书、课程、学费、资料、考试 | 学习 |
| 电影、游戏、会员、旅游、演唱会 | 娱乐 |
| 药、医院、门诊、体检 | 医疗 |
| 房租、水电、物业、宽带 | 住房 |

> 无法匹配时系统会提示或使用"其他"分类，用户仍可手动修改。

## 7. 退出登录

点击页面右上角"退出"按钮 → 确认后系统会清除本地登录信息并回到 `login.html`。

> 注意：在线模式下退出不会删除数据库中的任何数据；演示模式下，退出会同时清空本地的账单示例数据。

## 8. 常见问题（FAQ）

| 问题 | 解答 |
|------|------|
| 打开 login.html 输入 test/test123456 提示"后端未启动" | 直接使用演示模式，点击登录时会自动切换；也可以按第 3 节中的说明部署后端 |
| 登录后看不到任何账单 | 首次使用时系统没有账单数据，请到"账单"页手动添加 |
| 分类列表是空的 | 请确认后端数据库已正确执行 init.sql（包含 14 条默认分类） |
| 忘记密码怎么办 | 目前版本无找回密码功能，请联系系统管理员，或注册新账号 |
| 页面样式错乱 | 刷新页面；确认 styles.css 与 index.html 放在同一目录 |
| 修改账单后数据未更新 | 请刷新页面；一般情况下系统会自动重新拉取并刷新表格 |
| 预算设置后使用率是 0% | 如果当月没有账单，支出为 0，属于正常 |
| 想要更换月份统计 | 仪表盘和账单页都有月份选择器，选择任意月份即可 |
| 数据安全吗 | 密码经过 bcrypt 加盐哈希，不存在数据库的明文密码；所有请求需要 JWT 鉴权；账单数据按用户隔离 |

## 9. 快捷键与操作提示

（当前版本无快捷键，所有操作通过按钮与表单完成）

- 在"账单"表格的"操作"列点击编辑/删除
- 所有金额以人民币（￥）计，保留两位小数
- 日期时间可选填，默认当前时间
- 备注字段可选，长度不超过 255 个字符

## 10. 数据与备份

**在线模式**
- 数据存储在 MySQL 数据库 `ai_finance` 中
- 管理员可定期执行 `mysqldump` 进行数据备份
- 建议每月导出一次账单数据以防数据丢失

**演示模式**
- 数据保存在浏览器的 localStorage（键名：`ai_finance_data`）
- 清除浏览器数据或更换浏览器会导致数据丢失
- 建议重要数据使用在线模式

## 11. 错误信息说明

| 提示 | 原因 | 建议 |
|------|------|------|
| "后端服务未启动" | cargo run 没运行或运行在其他端口 | 启动后端服务 |
| "用户名或密码错误" | 账号不存在 / 密码输错 | 重新输入，或注册新账号 |
| "金额必须大于 0" | 输入了负金额或 0 | 把金额改为正数 |
| "分类类型与账单类型不匹配" | 例如用"工资"分类去记录支出 | 选择正确的分类 |
| "分类不存在" | 选择了非法的分类 ID | 从下拉菜单选择分类 |
| "无权操作" | 尝试修改/删除不属于自己的账单 | 检查账号是否正确 |
| "月份格式应为 YYYY-MM" | 月份输入错误 | 按"2026-06"这种格式输入 |
