# 项目开发计划

**项目名称**：AI 个人财务管理系统（AI Personal Finance Tracker）

---

## 1. 概述

### 1.1 项目目标

开发一个轻量的个人财务管理 Web 应用，提供：
- 支持用户注册、登录、安全退出
- 支持账单的增删改查、筛选、分页
- 支持分类管理
- 支持按月/按分类设定预算，实时跟踪预算使用率
- 提供月度收支统计和分类占比饼图
- 通过关键词规则自动识别账单分类
- 生成月度财务报告与建议

### 1.2 项目交付物

| 编号 | 交付物 | 形式 |
|------|--------|
| D1 | 后端 API 服务 | Rust 可执行程序（cargo 构建） |
| D2 | 前端应用 | HTML/CSS/JS 文件 |
| D3 | 数据库脚本 | `init.sql` |
| D4 | 软件文档 | 本目录下 8 份文档 |
| D5 | 测试账号 | test / test123456 |

### 1.3 关键里程碑

| 里程碑 | 完成时间 | 主要内容 |
|--------|--------|--------|
| M1 | 第 1 周 | 需求确认、数据库设计、后端框架搭建 |
| M2 | 第 2 周 | 鉴权模块、账单模块、分类与预算模块 |
| M3 | 第 3 周 | 统计与 AI 报告模块 |
| M4 | 第 4 周 | 联调、测试、文档整理 |

## 5. 质量保证计划

### 5.1 代码规范

| 层面 | 要求 |
|------|------|
| Rust | 使用 rustfmt / clippy，避免 unsafe |
| SQL | 禁止字符串拼接，强制参数绑定 |
| 前端 | 函数单份文件不超过 300 行 |
| 命名 | 统一使用驼峰、下划线、下划线命名规范 |

### 5.2 测试计划

| 测试类型 | 目标 | 负责人 |
