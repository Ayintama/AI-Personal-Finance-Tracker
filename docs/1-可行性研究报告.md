# 可行性研究报告

---

## 1. 项目概述

### 1.1 项目背景

随着个人可支配收入的增长和消费场景的多元化，居民个人财务管理的需求日益突出。传统的纸质记账和简单的电子表格难以提供及时、直观、可分析的财务视图。市面上已有各类记账应用，但普遍存在以下痛点：

- 部分产品收费或存在广告推送
- 数据隐私性难以保证（账单数据存放于第三方服务器）
- 缺乏对个人消费行为的智能分析与提醒

本项目旨在开发一款轻量、开源、可本地部署的个人财务管理 Web 应用，提供从记账到预算再到智能分析的完整闭环。

### 1.2 项目目标

- 提供简洁易用的浏览器端记账界面
- 支持对账单进行分类、筛选与分页浏览
- 支持按月或按分类设定预算并实时跟踪消费进度
- 提供月度收支统计与消费占比饼图
- 通过关键词规则自动识别账单分类
- 生成带建议的月度财务报告
- 保证用户数据的安全性与私密性（JWT 鉴权 + 密码哈希存储）

### 1.3 系统名称与代号

- 中文名称：AI 个人财务管理系统
- 英文名称：AI Personal Finance Tracker
- 项目代号：ai-finance

## 2. 技术可行性分析

### 2.1 技术选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 后端语言 | Rust 1.70+ | 内存安全、高性能，适合处理财务数据 |
| Web 框架 | Actix-web 4.x | 轻量级、异步、社区成熟 |
| 数据库 | MySQL 8.0 | 关系型数据库，支持事务，生态完整 |
| ORM 层 | SQLx 0.7 | 编译期类型检查，无运行时开销 |
| 身份认证 | JWT (jsonwebtoken) + bcrypt | 无状态鉴权，密码加盐哈希 |
| 前端 | HTML5 + CSS3 + 原生 JavaScript | 零依赖，浏览器直开即可用 |
| 架构模式 | 前后端分离，RESTful API | 解耦清晰，便于独立维护与部署 |

### 2.2 关键技术难点评估

1. **JWT 鉴权中间件**：需要在 Actix-web 的 wrap 层统一校验 Authorization 头。已实现 JwtMiddleware，使用 HS256 算法，Token 有效期 7 天。
2. **SQL 防注入**：所有参数使用 SQLx `?` 占位符绑定，禁止字符串拼接。
3. **金额精度**：使用 `DECIMAL(10,2)` 存储金额，避免浮点误差。
4. **离线/在线双模式**：前端支持 localStorage 兜底，便于开发期调试与演示。

### 2.3 结论

技术方案成熟、开源依赖均为广泛使用的稳定版本。一人团队即可完成，**技术上完全可行**。

## 4. 操作可行性分析

### 4.1 使用门槛

- 用户只需浏览器（Chrome / Edge / Firefox 任一）
- 注册用户 10 秒即可完成
- 记账操作：选择分类 → 输入金额 → 选时间 → 确认，三步骤完成

### 4.2 部署门槛

- 开发人员：安装 Rust 工具链 + MySQL + `cargo run`，约 10-20 分钟完成
- 非技术人员：可直接双击前端 HTML 进入演示模式（localStorage 版本）

### 4.3 结论

操作门槛极低，**操作上完全可行**。

## 6. 风险与对策

| 风险 | 发生概率 | 影响 | 对策 |
|------|---------|------|------|
| MySQL 密码泄露 | 中 | 高 | `.env` 文件不提交至 Git；生产环境强制改密 |
| JWT 密钥泄露 | 低 | 高 | 生产环境使用足够长随机字符串作为密钥 |
| 用户并发量高 | 极低 | 中 | Actix-web 支持多线程，必要时增加连接池容量 |
| 浏览器兼容性 | 低 | 低 | 仅使用标准 HTML5/ES6 特性，避免特定浏览器 API |
