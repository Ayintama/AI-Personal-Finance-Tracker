# 软件需求说明书（SRS）

**项目名称**：AI 个人财务管理系统


---

## 1. 引言

### 1.1 编写目的

本文档描述 AI 个人财务管理系统的功能与非功能需求，为开发、测试、验收提供统一依据。阅读对象为项目成员、测试人员及最终用户。

### 1.2 范围

本系统是一款基于 Web 的个人财务管理应用，面向个人用户使用。系统提供：

- 账户注册与登录
- 收支账单管理（增/删/改/查/分页）
- 分类管理
- 预算设置与进度跟踪
- 月度统计与消费占比
- 智能分类与 AI 报告

### 1.3 术语定义

| 术语 | 解释 |
|------|------|
| JWT | JSON Web Token，用于无状态身份认证 |
| bcrypt | 用于密码加盐哈希的算法 |
| 分类 | 账单所属类别（如餐饮、交通、工资等） |
| 预算 | 用户为某月设定的最大支出额 |
| 月度报告 | 基于某月数据生成的财务概览与建议 |

## 2. 总体描述

| 角色 | 描述 |

## 3. 功能需求（Functional Requirements）

### FR-1 用户认证

**描述**：用户需注册账号并登录后，才能操作个人账单数据。

| 子项 | 细节 |
|------|------|
| FR-1.1 | 用户可输入用户名 + 密码 + 邮箱（可选）完成注册 |
| FR-1.2 | 注册时用户名 3-50 字符，密码至少 6 位；邮箱需合法 |
| FR-1.3 | 用户名唯一，不可重复 |
| FR-1.4 | 登录时校验密码（bcrypt 比对），成功后返回 JWT Token |
| FR-1.5 | 未登录访问受保护接口，返回 401 并跳转到登录页 |
| FR-1.6 | 支持安全退出，清除本地 Token |

### FR-2 账单管理

**描述**：用户可对个人收支账单进行增删改查。

| 子项 | 细节 |
|------|------|
| FR-2.1 | 可新增账单，包含：分类（必填）、类型（收入/支出，必填）、金额（必填，>0）、发生时间、备注 |
| FR-2.2 | 可按月份筛选、按类型（收入/支出）筛选账单列表 |
| FR-2.3 | 账单列表分页显示，默认每页 20 条 |
| FR-2.4 | 可编辑账单（修改分类、金额、备注） |
| FR-2.5 | 可删除账单，但只能删除自己的账单 |
| FR-2.6 | 分类与类型必须匹配（例如账单类型是支出时，分类必须是支出类） |

### FR-3 分类管理

| 子项 | 细节 |
|------|------|
| FR-3.1 | 系统预置 14 个分类（8 个支出，6 个收入） |
| FR-3.2 | 分类按类型区分：支出型（expense）与收入型（income） |
| FR-3.3 | 用户可查看所有自己可用的分类（系统分类 + 自己添加的） |

### FR-4 预算管理

| 子项 | 细节 |
|------|------|
| FR-4.1 | 用户可为某个月设置总预算（category_id 为空） |
| FR-4.2 | 用户可为某个月的某个分类设置分类预算 |
| FR-4.3 | 同一用户同一月的同一分类不允许重复设置（自动覆盖旧值） |
| FR-4.4 | 查询时返回预算金额、已支出、剩余、使用率百分比 |

### FR-5 月度统计

| 子项 | 细节 |
|------|------|
| FR-5.1 | 统计指定月份的收入总额、支出总额、结余 |
| FR-5.2 | 返回分类 → 金额 的饼图数据（分类支出占比） |
| FR-5.3 | 返回该月总预算信息（含使用率、是否超预算） |

### FR-6 AI 报告

| 子项 | 细节 |
|------|------|
| FR-6.1 | 生成选定月份的文字报告，包含：本月收入、支出、结余、最大支出类别、预算进度 |
| FR-6.2 | 报告中至少包含 1-4 条建议（例如：超预算提醒、最大消费类别提醒、结余建议等） |
| FR-6.3 | 支持按备注 + 金额 + 类型进行自动分类（基于关键词规则） |
| FR-6.4 | 智能分类返回建议分类 id、分类名及置信度 |

### FR-7 前端 UI

| 子项 | 细节 |
|------|------|
| FR-7.1 | 仪表盘页展示本月收入、支出、结余、预算使用率、近期账单列表、分类占比饼图 |
| FR-7.2 | 账单页可按月份、类型过滤，支持新增/编辑/删除 |
| FR-7.3 | 预算页可设置总预算或分类预算，显示预算/已用/剩余/使用率 |
| FR-7.4 | AI 报告页可选择月份后生成报告与建议 |
| FR-7.5 | 所有操作有轻量 toast 提示 |

## 4. 非功能需求（Non-Functional Requirements）

### NFR-1 性能

| 指标 | 目标 |
|------|------|
| 单接口响应时间 | P95 < 500 ms（在 1000 条/月账单数据量下） |
| 并发用户 | 支持至少 10 个同时在线用户 |
| 首屏加载 | < 2 s |

### NFR-2 可用性

| 要求 | 说明 |
|------|------|
| 操作易用 | 3 次点击内完成一次记账 |
| 错误提示 | 操作失败有清晰中文提示 |
| 断网兼容 | 前端支持 localStorage 演示模式 |

### NFR-3 安全

| 要求 | 说明 |
|------|------|
| 密码存储 | bcrypt 加盐哈希，不存储明文 |
| 传输 | 生产环境应启用 HTTPS |
| 鉴权 | JWT Token，包含用户 id 和用户名 |
| 数据隔离 | 每个用户只能查看/编辑自己的账单/预算 |
| SQL 注入防护 | 所有查询使用参数绑定 |
| 输入校验 | 用户名长度、密码长度、邮箱格式、金额为正数等 |

### NFR-4 兼容性

| 浏览器 | 版本要求 |
|--------|---------|
| Chrome | 100+ |
| Firefox | 100+ |
| Edge | 100+ |

### NFR-5 可维护性

- 前后端分离，RESTful API 风格
- 模块划分清晰：routes、data、middleware、config、response
- 统一响应格式：`{ "code": 0, "message": "success", "data": ... }`

## 5. 接口需求

本系统对外暴露 RESTful API，所有响应为 JSON。详见项目内 `routes.rs` 代码文件。

| 方法 | 路径 | 功能 | 鉴权 |
|------|------|------|------|
| POST | `/api/auth/register` | 用户注册 | 否 |
| POST | `/api/auth/login` | 用户登录 | 否 |
| GET | `/api/auth/profile` | 获取当前用户信息 | 是 |
| GET | `/api/categories` | 获取分类列表 | 是 |
| GET | `/api/transactions` | 账单列表（分页+过滤） | 是 |
| POST | `/api/transactions` | 新增账单 | 是 |
| PUT | `/api/transactions/{id}` | 更新账单 | 是 |
| DELETE | `/api/transactions/{id}` | 删除账单 | 是 |
| GET | `/api/budgets` | 查询指定月份预算 | 是 |
| POST | `/api/budgets` | 创建/更新预算 | 是 |
| DELETE | `/api/budgets/{id}` | 删除预算 | 是 |
| GET | `/api/statistics/monthly` | 月度统计数据 | 是 |
| POST | `/api/ai/report` | 生成 AI 月度报告 | 是 |
| POST | `/api/ai/classify` | 根据备注智能分类 | 是 |

## 6. 验收标准

| 需求编号 | 验收条件 |
|---------|---------|
| FR-1 | 可用新注册账号登录；密码错误提示明确 |
| FR-2 | 能新增、编辑、删除账单；能按月份与类型筛选 |
| FR-3 | 分类列表返回正确类型（支出/收入） |
| FR-4 | 总预算、分类预算均可设置；重复设置自动覆盖 |
| FR-5 | 月度统计返回正确收入、支出、分类占比 |
| FR-6 | 报告生成至少 1 条建议；智能分类能识别"午餐" → "餐饮" |
| FR-7 | 四个主页面功能可交互，控制台无未捕获错误 |
| NFR-3 | 未登录访问 `api/transactions` 返回 401 |
