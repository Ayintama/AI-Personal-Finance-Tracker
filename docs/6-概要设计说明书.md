# 概要设计说明书

**项目名称**：AI 个人财务管理系统

---

## 1. 总体结构

### 1.1 系统架构

本系统采用经典的 **前后端分离**（Client-Server）架构：

```
┌──────────────────────┐         HTTPS / HTTP         ┌──────────────────────┐
│   浏览器前端 (Client) │──────────────────────────────▶│   后端 API 服务 (Web)│
│  HTML5 + CSS3 + JS   │          RESTful API          │  Rust + Actix-web    │
│                      │◀──────────────────────────────│  MySQL 8.0           │
└──────────────────────┘      JSON / JWT Token        └──────────────────────┘
                                                               │
                                                               ▼
                                                      ┌──────────────────────┐
                                                      │ 数据库 MySQL ai_finance│
                                                      │  users / categories/ │
                                                      │  transactions/budgets│
                                                      └──────────────────────┘
```

### 1.2 模块划分（后端）

| 模块 | 代码位置 | 职责 |
|------|---------|------|
| main | `src/main.rs` | 启动 Actix-web，装配中间件与路由，初始化数据库连接池 |
| config | `src/config.rs` | 读取环境变量、封装配置对象 |
| data | `src/data.rs` | 定义数据模型、封装 SQL 查询（数据访问层 DAO） |
| middleware | `src/middleware.rs` | JWT 认证中间件、Token 签发与解析 |
| response | `src/response.rs` | 统一响应格式、错误类型、错误码定义 |
| routes | `src/routes.rs` | 所有接口处理器（RESTful handler） |

### 1.3 模块划分（前端）

| 模块 | 文件 | 职责 |
|------|------|------|
| 登录页 | `frontend/login.html` + `common.js` | 注册/登录表单、提交 API、保存 Token |
| 主应用 | `frontend/index.html` | 4 个页面（仪表盘/账单/预算/报告）容器 |
| 业务逻辑 | `frontend/app.js` | 状态管理、数据渲染、交互事件绑定、API 调用 |
| 工具层 | `frontend/common.js` | fetch 封装、Token 管理、toast、格式化 |
| 样式 | `frontend/styles.css` | 页面视觉 |

### 1.4 业务模块清单

| 模块 ID | 模块名 | 功能简述 |
|---------|--------|----------|
| M-Auth | 用户认证 | 注册、登录、获取信息 |
| M-Category | 分类管理 | 获取用户可用分类列表 |
| M-Transaction | 账单管理 | 账单增删改查、分页与筛选 |
| M-Budget | 预算管理 | 设置/查询/删除预算 |
| M-Statistic | 统计 | 月度收支与分类占比 |
| M-AI | AI 报告 | 月度报告、智能分类 |

## 2. 模块间关系与调用关系

### 2.1 调用图

```
┌──────────────┐
│  routes.rs   │ ← HTTP 请求 (actix-web handler)
│  (接口层)    │ ─→ 1. 身份解析 (JwtMiddleware)
└──────┬───────┘   2. 参数校验
       │               3. 调用 data 层数据库函数
       │               4. 组装统一响应 (ApiResponse)
       ▼
┌──────────────┐
│  data.rs     │ ← 封装所有 MySQL 查询 (DAO)
│  (数据访问层)│ ─→ 使用 SQLx 参数绑定查询 MySQL
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  MySQL DB    │
└──────────────┘
```

### 2.2 模块依赖图（模块耦合关系）

```
main.rs
 ├── config.rs       (仅配置读取)
 ├── middleware.rs   (JWT 校验、依赖 jwt)
 ├── response.rs     (统一响应 / 错误)
 ├── data.rs         (DAO，依赖 sqlx)
 └── routes.rs       (handler，依赖 middleware/data/response)
```

**设计原则**：禁止循环依赖；单向依赖（上层依赖下层）。

## 4. 运行设计

### 4.1 启动流程

1. 读取 `.env` 中的数据库 URL、JWT 密钥、端口
2. 建立 MySQL 连接池（max_connections=10）
3. 创建 `AppState`（含 pool + jwt_secret）
4. 装配 Cors 中间件、Logger 中间件、JWT 中间件
5. 注册路由（`/health`、`/api/auth/*`、`/api/transactions/*` 等）
6. 绑定 `127.0.0.1:8080` 并监听

### 4.2 鉴权流程（请求生命周期）

```
请求到达
  ↓
Cors 预检处理
  ↓
JwtMiddleware 解析 Authorization: Bearer <token>
  ↓
解析失败 → 返回 401
解析成功 → 写入请求扩展（user_id）
  ↓
Handler 获取 user_id → 调用 data.rs 中的查询
  ↓
构造统一 ApiResponse 返回 JSON
```

### 4.3 异常处理策略

- 错误统一使用 `AppError` 枚举描述
- 错误经 Actix-web 的 Responder 实现转换为 HTTP 状态码
- 前端接收 `{ code, message, data }` 格式

## 5. 接口设计（核心）

| 方法 | 路径 | 请求体 | 返回 |
|------|------|--------|------|
| POST | /api/auth/register | `{ username, password, email? }` | `{ token, user }` |
| POST | /api/auth/login | `{ username, password }` | `{ token, user }` |
| GET | /api/transactions | Query: `month, type, page, page_size` | `{ items[], total }` |
| POST | /api/transactions | `{ category_id, amount, type, occurred_at, remark? }` | `{ id }` |
| PUT | /api/transactions/{id} | `{ category_id?, amount?, remark? }` | `{}` |
| POST | /api/budgets | `{ amount, month, category_id? }` | `{ id, month }` |
| GET | /api/statistics/monthly | Query: `month` | `{ income, expense, balance, category_totals, budget }` |
| POST | /api/ai/report | `{ month }` | `{ income, expense, balance, top_category, suggestions[], budget }` |
| POST | /api/ai/classify | `{ remark, amount, type }` | `{ category_id, category_name, confidence }` |

## 7. 系统安全设计

| 安全点 | 设计方案 |
|--------|---------|
| 密码存储 | bcrypt，cost=12，加盐哈希 |
| 身份认证 | JWT，HS256，7 天有效期 |
| 防 SQL 注入 | SQLx 参数绑定，禁止字符串拼接 |
| 跨站请求伪造 | 非浏览器端通过 Authorization 头承载 Token，不依赖 Cookie |
| CORS | Actix-cors 开放所有 Origin（开发期）；生产期改为指定 Origin |
| 数据隔离 | 所有查询强制 user_id 条件；删除/更新前二次校验 |

## 10. 异常与错误处理策略

| 错误情况 | HTTP 状态 | 前端行为 |
|---------|----------|---------|
| 未认证 | 401 | 跳 login.html |
| 参数非法 | 400 | 显示中文提示 |
| 无权操作（越权） | 403 | 提示无权 |
| 资源不存在 | 404 | 提示不存在 |
| 冲突（用户名重复） | 409 | 提示已存在 |
| 服务端错误 | 500 | 提示服务异常 |
