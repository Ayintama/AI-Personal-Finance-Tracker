# 测试计划

**项目名称**：AI 个人财务管理系统


---

## 1. 测试目的

验证软件是否符合《软件需求说明书》中的功能与非功能需求；发现代码缺陷；为验收与上线提供依据。

## 2. 测试范围

### 2.1 功能范围

- 用户认证模块（注册、登录、退出、未登录访问受保护接口）
- 账单管理模块（增删改查、月份筛选、类型筛选、分页）
- 分类模块（获取默认分类列表）
- 预算模块（设置、查询、删除、重复自动覆盖）
- 统计模块（月度收入/支出/结余、分类占比）
- AI 报告模块（月度报告、自动分类）

### 2.2 非功能范围

- 安全性：JWT 鉴权、bcrypt 密码哈希、SQL 注入防御
- 兼容性：Chrome、Edge、Firefox 三种主流浏览器
- 性能：主要接口响应时间 ≤ 500 ms

### 2.3 不测试范围

- 部署平台稳定性
- 底层数据库崩溃恢复
- 压力测试（超过 1000 用户并发）

## 4. 功能测试用例（核心）

> 每条用例格式：`编号 / 测试场景 / 步骤 / 预期输出`

### 4.1 认证模块

| 编号 | 场景 | 步骤 | 预期输出 |
|------|------|------|---------|
| TC-A1 | 成功注册新用户 | (1) POST /api/auth/register，填入合法的 username/password/email | code=0，返回用户 id |
| TC-A2 | 重复用户名注册 | (1) 先注册一个用户；(2) 再次用相同用户名注册 | code≠0，返回"用户名已存在" |
| TC-A3 | 密码太短 | (1) 密码输入 3 位字符 | 返回错误信息"密码至少 6 位" |
| TC-A4 | 登录成功 | (1) POST /api/auth/login，正确的用户名密码 | 返回 token + user 对象 |
| TC-A5 | 登录失败 | (1) 密码错误 | 返回 401 或 code≠0，提示"用户名或密码错误" |
| TC-A6 | 未登录访问账单 | (1) 直接 GET /api/transactions | 返回 401，前端跳转登录页 |
| TC-A7 | Token 过期后调用 | (1) 使用过期 Token 调用接口 | 返回 401 |

### 4.2 账单管理模块

| 编号 | 场景 | 步骤 | 预期输出 |
|------|------|------|---------|
| TC-T1 | 新增支出账单 | (1) 登录获取 token；(2) POST /api/transactions，填分类+金额+类型+时间 | 返回 id，列表中出现新账单 |
| TC-T2 | 金额为 0 时报错 | (1) POST amount=0 | 返回错误：金额必须大于 0 |
| TC-T3 | 账单分类与类型不匹配 | (1) 账单类型选"支出"，但分类 id 是"工资"（收入类） | 返回错误："分类类型与账单类型不匹配" |
| TC-T4 | 按月份筛选账单 | (1) 调用 `?month=2026-06` | 仅返回 6 月账单 |
| TC-T5 | 按类型筛选 | (1) 调用 `?type=expense` | 仅返回支出账单 |
| TC-T6 | 分页正常 | (1) 调用 `?page=1&page_size=5` | 返回指定页大小的账单 |
| TC-T7 | 更新自己的账单 | (1) PUT /api/transactions/{自己的 id} | 返回 200 且数据更新 |
| TC-T8 | 删除自己的账单 | (1) DELETE /api/transactions/{自己的 id} | 返回 200 且列表中消失 |
| TC-T9 | 越权删除他人账单 | (1) DELETE 不属于自己的 id | 返回 403（无权操作） |

### 4.3 预算模块

| 编号 | 场景 | 步骤 | 预期输出 |
|------|------|------|---------|
| TC-B1 | 设置 6 月总预算 | (1) POST /api/budgets，amount=3000，month=2026-06 | 返回新预算 id |
| TC-B2 | 同月同分类重复设置覆盖 | (1) 再次 POST 同样月份/分类 | 数据库中只有一条（覆盖式） |
| TC-B3 | 设置分类预算 | (1) POST /api/budgets，指定 category_id | 返回成功，列表中可见 |
| TC-B4 | 预算使用率统计 | (1) 设置 3000 总预算后，累计账单 1500 | 月度统计返回 50% 使用率 |
| TC-B5 | 删除预算 | (1) DELETE /api/budgets/{id} | 返回 200，列表中消失 |

### 4.4 统计与 AI 报告

| 编号 | 场景 | 步骤 | 预期输出 |
|------|------|------|---------|
| TC-S1 | 月度统计 | (1) GET `/statistics/monthly?month=2026-06` | 返回收入、支出、结余、分类占比 map、预算 |
| TC-S2 | 无账单月度统计 | (1) 查一个没有账单的月份 | income=0, expense=0 |
| TC-AI1 | 生成 AI 报告 | (1) POST /api/ai/report，指定月份 | 返回多条中文建议 |
| TC-AI2 | 智能分类识别 | (1) POST /api/ai/classify，remark="午餐" | 返回"餐饮"分类 |
| TC-AI3 | 智能分类 fallback | (1) POST 无法识别关键词 | 返回"其他"分类或同类 fallback |

## 7. 缺陷管理

### 7.1 缺陷分级

| 等级 | 含义 | 修复时限 |
|------|------|---------|
| P0 致命 | 导致系统崩溃、数据丢失、金钱错误 | 立即 |
| P1 严重 | 核心功能不可用（例如无法登录） | 24 h |
| P2 一般 | 功能可绕过、显示错误等 | 一周 |
| P3 轻微 | UI 文字、对齐等问题 | 下一轮 |

### 7.2 缺陷生命周期

```
新建 → 分析 → 分配 → 修复 → 回归测试 → 关闭
                      ↓（非问题/重复）→ 拒绝
```

## 9. 通过/失败准则

### 9.1 通过准则

- 所有 P0 / P1 缺陷已修复
- 核心用例（TC-A1、TC-A4、TC-T1、TC-T4、TC-B1、TC-B4、TC-S1、TC-AI1）全部通过
- 主要浏览器无布局错位
- 登录后前端无未捕获异常

### 9.2 失败准则

- 存在 1 个以上未解决 P0 缺陷
- 存在 3 个以上未解决 P1 缺陷
- 性能严重不达标（核心接口 > 2s）
