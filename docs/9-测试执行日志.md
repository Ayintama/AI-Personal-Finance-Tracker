### Bug 记录

| Bug ID | 严重度 | 描述 | 状态 |
|--------|--------|------|------|
| BUG-01 | P1 | `max_single_expense` 在无交易时因 MySQL MAX() 返回 NULL 导致 `ColumnDecode UnexpectedNullError` | ✅ 已修复 |
| BUG-02 | P1 | `trend_summary` 中 `start` 参数仅传月份字符串（如 `"2025-12"`），未拼接完整 DATETIME 格式，导致 MySQL 无法正确比较日期，查询返回空 | ✅ 已修复 |
| BUG-03 | P1 | 趋势接口 `months` 参数语义偏差——查询范围不含本月，传入 `months=6` 得到上月及往前 5 个月，漏掉当前月 | ✅ 已修复 |

**BUG-01 修复**：`MAX(amount)` → `COALESCE(MAX(amount), 0)`，`fetch_optional` → `fetch_one`。

**BUG-02 修复**：`months_ago(months)` → `format!("{}-01 00:00:00", months_ago(months))`。

**BUG-03 修复**：查询时传入 `months - 1`，使结果范围覆盖当前月份。

**修复方案**：将 `MAX(amount)` 改为 `COALESCE(MAX(amount), 0)`，`fetch_optional` 改为 `fetch_one`。
